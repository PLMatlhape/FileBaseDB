# Security Policy

## Supported versions

Security fixes are provided for the latest published version of FileBaseDB.

## Reporting a vulnerability

If you discover a security issue, please do not open a public issue containing exploit details.

Instead, report it privately through the repository owner’s preferred security contact or GitHub Security Advisories.

Please include:

- A clear description of the issue
- Affected version(s)
- Steps to reproduce
- Any logs or proof of concept necessary to validate the report

## What to avoid in reports

- Do not include real OAuth tokens or user data
- Do not upload private cloud files
- Do not share exploit details publicly until a fix is available

## Data handling reminder

FileBaseDB is designed to operate against the user’s own Google Drive or OneDrive account through official APIs. If you find a bug that could expose data beyond the intended account or folder boundaries, treat it as a high-priority security report.
